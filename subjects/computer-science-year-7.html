  • "What is the difference between hardware and software?"
    Answer: Hardware refers to the physical components you can touch (keyboard, mouse, screen, CPU), while software is the programs and applications that run on the computer (like Word, Chrome, or games). Hardware is the "body" of the computer, and software is the "brain" that tells it what to do.
  • "How can I protect my personal information online?"
    Answer: Protect personal information by: using strong, unique passwords; never sharing passwords with anyone; keeping privacy settings on social media accounts; not posting personal details like your address or phone number; being careful about what you share in photos; and thinking carefully before clicking links or downloading files.
  • "What is a digital footprint and why does it matter?"
    Answer: A digital footprint is the trail of information you leave behind when using the internet - posts, comments, photos, searches, and likes. It matters because this information can be seen by others (including future employers and universities), stays online for a long time, and shapes how people perceive you. Always think before you post!
  • "What is an algorithm?"
    Answer: An algorithm is a step-by-step set of instructions designed to solve a problem or complete a task. Like a recipe for baking a cake, an algorithm lists all the steps in the correct order. Computers follow algorithms to perform tasks, and good algorithms are clear, precise, and efficient.
  • "What are the three key programming concepts: sequence, selection, and iteration?"
    Answer: Sequence means instructions run in order (step 1, step 2, step 3). Selection means making decisions with "if/then" statements (if it's raining, take an umbrella). Iteration means repeating instructions multiple times using loops (repeat 10 times: move forward). These three concepts form the foundation of all programming.
  • "What does 'debugging' mean in programming?"
    Answer: Debugging is the process of finding and fixing errors (bugs) in your code. When a program doesn't work as expected, programmers debug by carefully checking each line, testing different parts, and making corrections. It's like being a detective - looking for clues to solve the mystery of why the code isn't working!

I can now answer... ü§î

  • "What is a variable in Scratch programming?"
    Answer: A variable is a container that stores information that can change, like a box with a label. In Scratch, you might use a variable called "score" to keep track of points in a game, or "lives" to track how many chances a player has left. The value can change throughout the program (increase, decrease, or be set to different numbers).
  • "How does a 'forever loop' work in Scratch?"
    Answer: A forever loop continuously repeats the code blocks inside it without stopping (unless you use a stop command or the program ends). It's perfect for things that need to keep happening throughout a game, like checking if a sprite touches another sprite, keeping the background music playing, or constantly checking for keyboard input.
  • "What does an 'if/then' conditional statement do?"
    Answer: An if/then statement checks if something is true, and only runs the code inside if the condition is met. For example: "IF the sprite touches the edge, THEN bounce back." This allows programs to make decisions and respond differently based on what's happening, making games and programs interactive and smart.
  • "What is a cell reference in a spreadsheet?"
    Answer: A cell reference is the address of a cell in a spreadsheet, using the column letter and row number (like A1, B5, C12). Cell references are used in formulas to perform calculations. For example, "=A1+B1" adds the values in cells A1 and B1. This makes spreadsheets powerful because formulas automatically update when cell values change.
  • "What is the difference between a formula and a function in spreadsheets?"
    Answer: A formula is a calculation you create yourself using operators like + or * (example: =A1+B1). A function is a pre-built formula that performs a specific task, like SUM, AVERAGE, or MAX (example: =SUM(A1:A10) adds all numbers from A1 to A10). Functions save time by doing complex calculations with simple commands.
  • "Why should I use charts and graphs in spreadsheets?"
    Answer: Charts and graphs visualize data, making patterns and trends easier to understand at a glance. Numbers in a table can be hard to interpret, but a bar chart or pie chart shows comparisons clearly. Different chart types work best for different data: bar charts for comparisons, line graphs for trends over time, and pie charts for showing parts of a whole.

I can now answer... ü§î

  • "What is a micro:bit and what can it do?"
    Answer: A micro:bit is a small programmable computer (about the size of a credit card) with a 5x5 LED display, two buttons, and built-in sensors including accelerometer, compass, and temperature sensor. You can program it to create games, display animations, measure temperature, count steps, and connect to other devices. It demonstrates how computers control real-world devices.
  • "How do inputs and outputs work on the micro:bit?"
    Answer: Inputs are ways to give information to the micro:bit, like pressing buttons A or B, shaking it, or tilting it. Outputs are ways the micro:bit responds, like displaying patterns on the LED screen or playing sounds. Programs connect inputs to outputs: "WHEN button A is pressed, THEN display a smiley face."
  • "What is the accelerometer sensor used for?"
    Answer: The accelerometer detects movement and orientation. It can sense when the micro:bit is shaken, tilted, turned upside down, or moved in different directions. This allows you to create programs like step counters, spirit levels, motion-controlled games, or shake-to-reset functions. It's the same sensor found in smartphones and fitness trackers.
  • "What is 'decomposition' in computational thinking?"
    Answer: Decomposition means breaking down a complex problem into smaller, more manageable parts that are easier to solve. Instead of trying to program an entire game at once, you might decompose it into: create the character, add movement controls, design obstacles, implement scoring, add sound effects. Solving each small part is easier than tackling everything together.
  • "What is 'abstraction' and why is it important?"
    Answer: Abstraction is simplifying a problem by focusing on essential information and hiding unnecessary details. For example, when using a "move forward" block in Scratch, you don't need to know the complex mathematics behind sprite positioning - you just use the simple command. Abstraction makes complex systems easier to understand and work with by focusing on what's relevant.
  • "How has my computing knowledge grown this year?"
    Answer: From understanding basic computer components and staying safe online, to creating Scratch games with loops and variables, managing data in spreadsheets with formulas and charts, and programming physical devices with micro:bit. You've developed computational thinking skills including algorithms, decomposition, abstraction, and debugging - preparing you for text-based programming (Python) in Year 8!
